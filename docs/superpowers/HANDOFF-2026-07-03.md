### 關鍵技術決策（已定）
- **連線本質**：不是 PG tunnel，而是 SSH 到主機後 `docker exec -i <container> psql --csv`（docker mode）或 `sudo -u <user> psql`（local mode）。SQL 經 base64 編碼防注入。需 npm `ssh2`（對應 paramiko）。
- **密碼儲存**：SSH 密碼用 **AES-256-GCM 對稱加密**（`APP_SECRET` 環境變數衍生金鑰）存 v2 DB，查詢時解密。與登入密碼的 pbkdf2（單向雜湊）不同——SSH 密碼必須可還原才能用，故用可逆的對稱加密。
- **SELECT-only 驗證**：移植原 python `_validate_select_only`（去尾分號、禁多語句、首字須 SELECT/WITH、剝字串註解後擋 DML/DDL）。
- **AI endpoint** `/ai/db/*`：只綁 loopback（`req.socket.remoteAddress`，不信任 header）、免登入；人用 route 需 verifyToken。
- **getSQL skill** 改呼叫 `http://localhost:3939/ai/db/connections?project=<name>` 與 `/ai/db/query`。

### 10 個 task 順序
1. DB migration（`db_connections` 表）+ 裝 ssh2
2. `lib/crypto.js`（AES-256-GCM encrypt/decrypt）
3. `lib/ssh-sql.js` 的 `validateSelectOnly`

8. 前端 `ProjectDbQuery.js` + 路由
9. 專案頁分頁式 topbar + 設定頁瘦身
10. 更新 getSQL skill SKILL.md
